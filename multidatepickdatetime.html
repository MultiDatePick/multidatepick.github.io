multiDatePickDateTime

Schedule Dates and Times in Salesforce Flows. Combine multi-date selection with per-date time picking for appointments, sessions, and shifts.

Install from AppExchange
🕐 Component Screenshot / GIF — multiDatePickDateTime in action Component-multiDatePickDateTime-1024x1024.jpg

Overview

multiDatePickDateTime extends the date selection calendar with time picking. After selecting dates, users assign a start time (and optionally an end time) to each one using dropdown menus or a visual time slot grid. You control the time interval (15, 30, or 60 minutes), the visible time range (e.g., 8:00 AM to 5:00 PM), and whether each date can have a different time or all dates share the same time.

Use this component when your Flow or Record Page needs both a date and a time — for appointment scheduling, session booking, shift planning, or any workflow where the time of day matters.

Getting Started: Step-by-Step Setup

Follow these steps to get multiDatePickDateTime working in your org. The child object setup is similar to multiDatePickDates, but with additional Time fields.

1 Create the Child Object (for Record Pages)

If you plan to use this on a Lightning Record Page with auto-save, you need a child object. If you only plan to use it in Flows, skip to Step 3.

Go to Setup > Object Manager > Create > Custom Object. For this example, we'll create an Appointment__c object. Name yours whatever fits (Session__c, Shift__c, Schedule_Entry__c, etc.).

🖼️Screenshot: Creating the Appointment object in Setup

2 Create the Fields on Your Child Object

Your child object needs a Date field, a relationship field, and optionally one or two Time fields for start and end times:

  • Date field (e.g., Appointment_Date__c) — stores the selected date
  • Lookup field (e.g., Contact__c) — links back to the parent record
  • Start Time field (e.g., Start_Time__c, type: Time) — stores the start time
  • End Time field (e.g., End_Time__c, type: Time) — stores the end time (optional)
🖼️Screenshot: Fields on the child object including Time fields

3 Add the Component to a Record Page or Flow

Same process as multiDatePickDates — drag the component onto a Record Page in App Builder or onto a Screen element in Flow Builder. Configure the properties below.

Using on a Lightning Record Page

multiDatePickDateTime supports all the same Record Page auto-save properties as multiDatePickDates (Related Object, Date Field, Relationship Field, Preload, Blocked Dates). See the multiDatePickDates page for those. The following additional properties are available for time fields:

Record Page Time & Save Properties

PropertyTypeDefaultDescription
startTimeFieldString—API name of the field on the related object where the selected start time will be saved. Example: Start_Time__c. Leave blank if your object has no time fields.
endTimeFieldString—API name of the field on the related object where the selected end time will be saved. Example: End_Time__c. Only used when Enable End Time is ON. Leave blank if not needed.
saveButtonLabelStringSaveText on the save button.
enableConflictCheckBooleanfalseWhen ON, checks for time conflicts against existing records in the configured Related Object. conflictLookAheadDaysInteger180How many days ahead to check for time conflicts when Enable Conflict Checking is ON. Only selected dates within this window are queried from Salesforce. Default: 180 (6 months). Reduce for better performance on high-volume objects (e.g., 30 or 60). Dates beyond this window are shown as available without a conflict check.
skipConflictsOnSaveBooleanfalseOnly applies when Time Display Mode is 'grid' and Allow Different Times Per Date is OFF (shared grid mode). When ON, each selected date is checked for time conflicts at save time. Dates with conflicts are skipped (not saved) and shown with a RED strikethrough on the calendar. Non-conflicting dates are saved normally and shown with a grey strikethrough. Requires: Related Object API Name, Date Field API Name, Start Time Field API Name, and End Time Field API Name. ⚠️
Additive Save: The component uses the additive save method on Record Pages — it inserts new date records with their associated times but does not delete existing records. This allows multiple records per date (e.g., two different appointment times on the same day).

Using the MultiDatePickParser in Your Flow

The MultiDatePickParser is an Apex invocable action included in the package. It takes the JSON output from the component and converts it into a collection of MultiDatePickEntry objects that you can loop through in your Flow. For DateTime, the key difference is that you pass selectedDatesWithTimesJson (not selectedDateRangesJson), and each MultiDatePickEntry will have startTime and endTime populated.

Step-by-Step Flow Setup

  1. Add the multiDatePickDateTime component to a Screen element. Turn ON "Enable End Time" if you need both start and end times. Wire selectedDatesWithTimesJson to a Text variable (e.g., {!dateTimesJson}).
  2. After the Screen element, add an Action element. Search for "Parse Date Entries from JSON". Pass {!dateTimesJson} as the input.
  3. Store the output in a variable of type Apex-Defined, class: MultiDatePickEntryCollection (e.g., {!parsedCollection}).
  4. Add a Loop element. Set the collection to {!parsedCollection.data}. This loops through each MultiDatePickEntry.
  5. Inside the loop, access: {!currentEntry.fromDate}, {!currentEntry.startTime}, {!currentEntry.endTime}.

MultiDatePickEntry Fields

FieldTypeDescription
idIntegerSequential identifier for each entry (1, 2, 3...)
fromDateDateThe date for this entry
toDateDateEnd date (same as fromDate for DateTime entries)
startTimeTimeStart time assigned to this date
endTimeTimeEnd time assigned to this date (populated when Enable End Time is ON)

💡
Tip: Unlike multiDatePickDates where you enable "Group Consecutive Dates as JSON Ranges", the DateTime component automatically populates selectedDatesWithTimesJson with time data. Just wire that output to a Text variable and pass it to the parser.

Example Use Cases

Here are real-world examples of how the MultiDatePickParser works with multiDatePickDateTime to create records with both dates and times:

1 Staffing & Recruiting — Shift Assignment

A recruiting agency is placing a worker at a client site. A staffing coordinator opens a Flow on the Placement__c record, uses multiDatePickDateTime with allowDifferentTimes ON and enableEndTime ON to select the worker's upcoming shift dates and assign a start and end time for each one. The Flow passes selectedDatesWithTimesJson to MultiDatePickParser. The parser returns a collection where each entry has fromDate, startTime, and endTime. A Loop creates one Shift__c record per entry, setting Shift_Date__c, Start_Time__c, and End_Time__c, and linking each record back to the Placement__c via a lookup. The worker's placement status is updated to "Shifts Assigned" after the loop completes.

Key fields used: fromDate, startTime, endTime

Pattern: Screen (DateTime) → MultiDatePickParser → Loop → Create Shift Records → Update Placement Status

2 Education — Instructor Session Scheduling

An instructor sets up tutoring or office hours sessions for a course across multiple weeks. Each session falls on a different day and at a different time. Using multiDatePickDateTime with allowDifferentTimes ON, the instructor picks dates and assigns a unique start and end time to each one. The Flow parses selectedDatesWithTimesJson, then loops through the collection to create Session__c records linked to the Course__c — each record capturing Session_Date__c, Start_Time__c, and End_Time__c. The Course__c status is updated to "Sessions Scheduled" after all records are created.

Key fields used: fromDate, startTime, endTime

Pattern: Screen (DateTime) → MultiDatePickParser → Loop → Create Session Records → Update Course Status

3 Medical/Healthcare — Patient Follow-Up Appointments

A care coordinator schedules a series of follow-up appointments for a patient after a procedure. Each follow-up is on a different date and at a specific time. Using multiDatePickDateTime with allowDifferentTimes ON, the coordinator picks the dates and assigns a time to each. The Flow passes selectedDatesWithTimesJson to MultiDatePickParser and loops through the entries to create Appointment__c records linked to the patient's Case — setting Appointment_Date__c, Start_Time__c, and End_Time__c on each. The Case status is updated to "Follow-Ups Scheduled" after the loop completes.

Key fields used: fromDate, startTime, endTime

Pattern: Screen (DateTime) → MultiDatePickParser → Loop → Create Appointment Records → Update Case Status
