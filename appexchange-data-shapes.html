One Picker. Five Data Shapes.

The same MultiDatePick selection can save five different ways — match the shape to your object model.

SHAPE 1

One Record per Date

Every selected date becomes its own record. Pick five dates, get five records.

Example use casesRSVPs, attendance tracking, daily check-ins, PTO request days.

SHAPE 2

One Record per Date Range

Adjacent picks collapse into a single record with a start date and an end date.

Example use casesPTO blocks, travel itineraries, multi-night stays, equipment rentals, vehicle reservations.

SHAPE 3

One Record per Time Block

Each contiguous block of time on each selected date becomes its own record with a start and end time.

Example use casesDaily timesheets, shift segments, billable time entries, multi-session events.

SHAPE 4

One Parent + Sub-Blocks for Gaps

A single parent record covers the full envelope. Every gap between picks becomes a child sub-block on a related object.

Example use casesTraining sessions with coffee breaks, field service visits with travel between stops, compliance-mandated rest periods inside a shift.

SHAPE 5

One Record per Resource × Date Block

Each selected resource × each selected date × each contiguous time block becomes its own record. Records multiply across the resource axis AND the time axis.

Example use casesMulti-room conferences, parallel team meetings, technician routes (one record per stop), shared resource bookings.
