multiDatePickDates

Select Multiple Dates in Salesforce Flows and Lightning Pages. The core date selection component — click, drag, or use recurring patterns to pick dates.

Install from AppExchange

Overview

multiDatePickDates is the core date selection component. It gives your users a full calendar interface where they can click individual dates, drag to select ranges, or use recurring patterns to select multiple dates at once. It works in Salesforce Screen Flows (outputting dates to Flow variables) and on Lightning Record Pages (saving dates as related records automatically).

Getting Started: Step-by-Step Setup

Follow these steps to get multiDatePickDates working in your org. This covers both Record Page and Flow usage.

1 Create the Child Object (for Record Pages)

If you plan to use multiDatePickDates on a Lightning Record Page with auto-save, you need a child object to store the selected dates. Each selected date becomes one record in this object. If you only plan to use it in Flows, skip to Step 3.

Go to Setup > Object Manager > Create > Custom Object. For this example, we'll create a Travel_Date__c object to store dates related to a Travel__c parent record. You can name yours whatever fits your use case (Appointment_Date__c, Project_Milestone__c, etc.).

🖼️Screenshot: Creating the child object in Salesforce Setup

2 Create the Fields on Your Child Object

Your child object needs exactly two custom fields:

  • Date field (e.g., Date__c) — stores the selected date
  • Lookup or Master-Detail field (e.g., Travel__c) — links back to the parent record
🖼️Screenshot: Fields on the child object

3 Add the Component to a Record Page

Open the parent record page (e.g., Travel__c) in App Builder. Drag multiDatePickDates from the custom components panel onto the page. Fill in the properties listed in the Record Page Properties section below.

Save and activate the page. That's it — users can now select dates on the calendar and click Save to create date records.

🖼️Screenshot: Component on a Record Page in App Builder

4 Add the Component to a Screen Flow (Alternative)

If you are using multiDatePickDates in a Flow instead of (or in addition to) a Record Page:

  1. Open Flow Builder and create or edit a Screen Flow.
  2. Add a Screen element and drag multiDatePickDates from the component palette onto the screen.
  3. Set the Button Label and Calendar Title to match your use case.
  4. Create a Text Collection variable (e.g., {!selectedDatesOutput}) and wire it to the "Selected Dates (Output)" property.
  5. Optionally turn ON "Group Consecutive Dates as JSON Ranges" and wire the JSON output to a Text variable, then use the MultiDatePickParser action after the screen to loop through entries.
  6. No objects or fields are needed for Flow-only usage — the component outputs dates to your Flow variables and you decide what to do with them.

To use multiDatePickDates in a Flow, add a Screen element and drag the component from the custom component palette onto your screen. Configure the properties below in the Flow Builder sidebar.

Using on a Lightning Record Page

To use multiDatePickDates on a Record Page, open the page in App Builder, drag the component from the custom components panel, and configure the properties below. When configured with auto-save fields, the component automatically creates related records as users select dates.

⚠️
Important: The component uses an additive save method — it creates new records for newly selected dates but does not delete existing records when dates are deselected. Users must delete unwanted records directly on the related list. This prevents accidental data loss.

All Display and Constraint properties from the Flow section above are also available on Record Pages. The following are additional properties specific to the Record Page context:

Record Page Auto-Save Properties

PropertyTypeDefaultDescription
saveButtonLabelStringSaveText on the save button. Example: 'Save Dates', 'Confirm Selection'
relatedObjectApiNameString—API name of the object where each selected date is saved as a record. Example: Travel_Date__c or Event. Each selected date creates one record in this object.
dateFieldApiNameString—API name of the Date field on the related object that stores the selected date. Example: Date__c or ActivityDate
relationshipFieldApiNameString—API name of the lookup field on the related object that links back to the current record. Example: Travel__c (lookup from Travel_Date__c to the parent Travel record)
preloadExistingDatesBooleanfalseWhen ON, queries existing related date records and shows them in the calendar. Requires Related Object, Date Field, and Relationship Field to be configured.
preloadModeStringeditableControls how existing dates appear. 'editable': dates load selected and can be changed. 'readonly': dates appear greyed out and cannot be removed. Example: editable
blockedDatesSourceObjectString—API name of an object to query for dates that should be blocked/unavailable. Example: Event (blocks calendar days that already have Events on this record).
blockedDatesDateFieldString—API name of the Date field on the blocked dates object. Example: ActivityDate
blockedDatesFilterFieldString—Optional: API name of a lookup field on the blocked object to filter results by the current record. Leave blank to block these dates globally. Example: WhoId

Using the MultiDatePickParser in Your Flow

The MultiDatePickParser is an Apex invocable action included in the package. It takes the JSON output from the component and converts it into a collection of MultiDatePickEntry objects that you can loop through in your Flow.

Step-by-Step Flow Setup

  1. Add the multiDatePickDates component to a Screen element. Turn ON "Group Consecutive Dates as JSON Ranges". Wire the Date Ranges JSON Output to a Text variable (e.g., {!dateRangesJson}).
  2. After the Screen element, add an Action element. Search for "Parse Date Entries from JSON". Pass {!dateRangesJson} as the input.
  3. Store the output in a variable of type Apex-Defined, class: MultiDatePickEntryCollection (e.g., {!parsedCollection}).
  4. Add a Loop element. Set the collection to {!parsedCollection.data}. This loops through each MultiDatePickEntry.
  5. Inside the loop, access the current item's fields: {!currentEntry.fromDate}, {!currentEntry.toDate}, {!currentEntry.startTime}, {!currentEntry.endTime}, {!currentEntry.id}.

MultiDatePickEntry Fields

FieldTypeDescription
idIntegerSequential identifier for each entry (1, 2, 3...)
fromDateDateStart date of the range
toDateDateEnd date of the range (same as fromDate for single dates)
startTimeTimeStart time (only populated by multiDatePickDateTime)
endTimeTimeEnd time (only populated by multiDatePickDateTime)

MultiDatePickEntryCollection Fields

FieldTypeDescription
dataList<MultiDatePickEntry>The list of date entries. Use this as the collection in your Flow Loop element.
totalCountIntegerNumber of entries in the collection.
successBooleanWhether the parsing was successful.
errorMessageStringError details if parsing failed (null on success).

Example Use Cases

Here are real-world examples of how the MultiDatePickParser, MultiDatePickEntry, and MultiDatePickEntryCollection work together in production Flows:

1 Travel & Expense — Create Per-Diem Records for Each Date Range

An employee selects their travel dates using multiDatePickDates with JSON output enabled. The Flow passes the JSON to MultiDatePickParser, then uses a Transform element to map each MultiDatePickEntry to a Travel_Details__c record — setting fromDate → Travel_Date__c and linking it back to the parent Travel_Request__c via a lookup. Each parsed entry becomes one travel detail line item.

Key fields used: fromDate, toDate, id

2 Maintenance Scheduling — One Schedule Record Per Date Range

A maintenance manager selects multiple maintenance windows (e.g., "March 10–12" and "March 17–19") on an asset record. The Flow parses the JSON into a MultiDatePickEntryCollection, uses a Transform element to map fromDate → Maintenance_Start_Date__c and toDate → Maintenance_End_Date__c, then loops through to create one Maintenance_Schedule__c record per date range. The asset status is updated to "Under Maintenance" after all records are created.

Key fields used: fromDate, toDate, id

Pattern: Parser → Transform → Loop → Create Records → Update Parent Status

3 Training Session Scheduling — Multi-Week Session Blocks

A training coordinator selects date ranges for a multi-week training program (e.g., "Week 1: April 7–11" and "Week 2: April 14–18"). The Flow parses the ranges, maps them to Training_Schedule__c records using fromDate → Session_Start_Date__c and toDate → Session_End_Date__c, and auto-generates a record name from the date range plus the training title. The parent Training_Session__c status is updated to "Scheduled".

Key fields used: fromDate, toDate, id

Pattern: Parser → Transform → Loop → Create Records → Update Parent Status

4 Validation & Error Handling — Using Collection Metadata

After calling MultiDatePickParser, check the MultiDatePickEntryCollection.success field in a Decision element before proceeding. If success is false, display the errorMessage to the user and route them back to the date selection screen. Use totalCount to show a confirmation message like "You selected {totalCount} date ranges — continue?" before creating records.

Key fields used: success, errorMessage, totalCount, data
